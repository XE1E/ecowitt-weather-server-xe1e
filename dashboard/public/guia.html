01 — Proyecto

El proyecto

Es un proyecto personal sobre el clima: una estación meteorológica propia, instalada en un punto exacto de la Ciudad de México (Benito Juárez), que publica en tiempo casi real las condiciones de ese lugar y las conserva para consultar su histórico, estadísticas y climatología.

Lo que lo distingue es que todo está hecho a la medida: el servidor que recibe los datos, la base de series de tiempo, la API y el sitio web se desarrollaron específicamente para esta estación y viven en un VPS propio —no una plataforma comercial «de caja»—. Eso permite decidir qué se mide, cómo se calcula y cómo se presenta, y hacer crecer la plataforma a voluntad.

  • Objetivo principal: aprovechar al máximo el dato local de la estación —real, cercano y del sitio exacto, no la interpolación de un modelo— y, a la par, desarrollar y evolucionar una plataforma propia (servidor + API + web) hecha a la medida en el VPS, además de compartir el dato con redes públicas.
  • Alcance: meteorología en sentido amplio —condiciones actuales, histórico, estadísticas y climatología, pronóstico, radar y satélite— más los campos vecinos que enriquecen la lectura del cielo: astronomía (sol, luna, almanaque), calidad del aire (AQI e IMECA), meteorología aeronáutica (METAR/TAF) y sismos de la región.
  • Más de un sitio: además de la estación principal, el servidor admite una estación remota (p. ej. un GW1100 en otra ubicación) que envía al mismo VPS; sus datos se guardan por separado y tienen su propia página, para comparar el clima de dos puntos distintos.

El sitio tiene dos vistas: / — vista clásica (tablero simple de un vistazo, unificado con el estilo de /pro) — y /pro — vista completa, con sus secciones, unidades conmutables, tema claro/oscuro y efectos de clima; instalable como app (PWA).

02 — Hardware

Hardware

Kit Ecowitt WS2910 + sensor WS69 + termohigrómetros WN31.

EquipoQué esQué mide / hace
Consola WS2910Pantalla + puente Wi-FiPresión (barómetro), temp/humedad interior; envía todos los datos al servidor por Wi-Fi (protocolo Ecowitt)
WS69 (7-en-1)Sensor exterior integradoTemp y humedad exterior, viento y dirección, ráfaga, lluvia (tasa/evento/día…), radiación solar e índice UV
WN31 (×8)Termohigrómetros de canalTemperatura y humedad en hasta 8 canales independientes
GW1100 (remota, opcional)Gateway Wi-Fi EcowittEstación secundaria: envía al mismo servidor; sus lecturas se guardan aparte y se ven en su propia página (solo lectura). No dispara alertas ni publica a redes

Baterías: la WS69, la consola y cada canal WN31 reportan su estado (OK / baja); el sistema avisa cuando alguna está baja. Envío: la consola se configura en Weather Services → Customized con protocolo Ecowitt hacia clima.xe1e.net, ruta /data/report/, una lectura cada ~16–60 s.

El hardware llega ~2026-07-17. Hasta entonces el sitio puede correr con un simulador; al instalar la estación se apaga el simulador, se limpian los datos falsos y se apunta la consola al servidor.

Arquitectura y flujo de datos

WS2910 + WS69 + WN31
     │  push HTTP (protocolo Ecowitt)
     ▼
Cloudflare (HTTPS) → Caddy (TLS) → Dashboard (React, sirve web + proxy /api)
                                        │
                                        ▼
                              Receiver (FastAPI) → InfluxDB 2.7 (histórico)
                                        ├─ MQTT / Home Assistant (opcional)
                                        ├─ Alertas (Telegram / log)

Componentes (contenedores Docker): receiver (FastAPI) recibe, procesa, guarda y expone la API; influxdb guarda el histórico; dashboard (React) sirve la web y hace de proxy de /api; caddy gestiona TLS con certificado Origin; Cloudflare hace de proxy y protección delante del VPS.

04 — Procesamiento

Procesamiento del dato local

Cada lectura pasa por un pipeline (inspirado en WeeWX) antes de guardarse:

parsear → convertir a métrico → calibrar → QC rangos → QC picos → derivar
        → guardar (InfluxDB) → MQTT → alertas → publicar a redes
  • Conversión a métrico: °F→°C, inHg→hPa, in→mm, mph→km/h.
  • Calibración (opcional): corrige sesgos del sensor con offsets (temp/humedad/presión) y multiplicadores (viento/lluvia).
  • QC por rangos: descarta valores físicamente imposibles (temp −80 °C, humedad 150 %, presión absurda para CDMX…).
  • QC por picos: descarta un valor que salta de forma imposible respecto a la lectura anterior (glitch); no filtra viento ni lluvia y se omite si la previa tiene más de 15 min.
  • Variables derivadas: punto de rocío, sensación térmica, índice de calor, sensación por frío/viento, humidex y base de nubes.
  • Guardado: en InfluxDB, más el resumen diario («Dayfile»): un registro por día con mín/máx/prom/total y la hora de cada extremo.

Todos estos ajustes se editan en caliente desde el panel, sin reiniciar.

05 — Sitio web

La página web (/pro)

Barra superior: título y ubicación, reloj en vivo, botón de unidades (métrico ↔ imperial), botón de tema (☀️/🌙), botón FX (efectos de clima animados), enlace a la Vista clásica y el cintillo de secciones. Tema y unidades se recuerdan entre visitas.

Inicio

Estado actual: mini-estadísticas (máx de hoy, viento, precipitación, UV, sensación, presión, prob. de lluvia, humedad, «vs ayer»); condiciones actuales (temp + ícono + sensación, punto de rocío, bulbo húmedo, humidex, base de nubes, con humedad/UV/presión en cajas); viento (tarjeta que gira a la rosa de vientos al pulsar); presión (mín/máx/tendencia y gráfica de 24 h con hora); pronóstico local (por tendencia del barómetro); pronóstico Open-Meteo; gráfica de temperatura; precipitación; UV/solar; sol y luna; calidad del aire (AQI) e IMECA; último sismo; alertas; próximos eventos; e interior y sensores adicionales WN31. Abajo, el radar.

Mi tablero

Una vista personalizable: con el botón «Personalizar» eliges qué tarjetas mostrar u ocultar (las mismas del Inicio) y, con ese modo activo, puedes arrastrarlas para reordenarlas a tu gusto (con soporte táctil). Tanto la selección como el orden se guardan en tu dispositivo. La tarjeta de condiciones actuales queda fija arriba. Entre las disponibles está la de la Estación remota (resumen compacto de la segunda estación).

Pronóstico

Pronóstico de Open-Meteo (modelo best_match) en pestañas por día y por hora, con tarjeta resumen del día y descripciones en lenguaje natural (NLG), más la explicación de qué son los modelos y por qué el pronóstico es probabilidad, no certeza.

Historia

El archivo de la estación con granularidad Día / Mes / Año y cinco grupos de gráficas interactivas (temperatura; viento con dirección; humedad y punto de rocío; radiación UV/solar; y precipitación y presión), tabla de días/meses clicable, detalle diario y exportación a CSV.

Estadísticas

Con selector de año: resumen del año, promedios mensuales, contadores de días (cálidos, noches frescas, con/sin lluvia), grados-día y evapotranspiración, récords históricos en pestañas por categoría (cada uno con su top 5 y fecha), estadísticas por periodo y una rosa de vientos.

Climatología

Climatología Local a partir del resumen diario: tarjetas de ayer / mes / año (temperatura, humedad, viento, lluvia, grados-día y ET), un climograma (lluvia + temperatura por mes), récords por mes, el reporte estilo NOAA (diario/mensual/anual, base 18.3 °C) y «en este día» (años previos).

Radar y satélite

Mapa interactivo de Ventusky (radar, nubes, viento, temperatura…) e imagen satelital diaria de NASA GIBS (color real, con selector de capa y fecha), ambos centrados en la estación.

Astronomía

Sol y Luna con arco de trayectoria y estadísticas (elevación, azimut, iluminación, edad, distancia…), fila de fases lunares, y el almanaque (pyephem, local): los tres crepúsculos (civil −6°, náutico −12°, astronómico −18°) y los planetas visibles (Mercurio…Saturno) con orto/ocaso, altitud y magnitud.

Calidad del aire

AQI (escala US EPA, vía WAQI/aqicn) e IMECA estimado con las tablas oficiales de la norma NADF-009-AIRE-2017 a partir de concentraciones modeladas (Open-Meteo/CAMS): valor y categoría con color, medidor visual, sub-índices por contaminante, recomendaciones de salud, pronóstico del IMECA por horas y aviso de contingencia. El AQI usa un token gratuito de WAQI (se pone en el panel).

Aeronáutica

METAR (observación) y TAF (pronóstico) decodificados al español, con la categoría de vuelo (VFR/MVFR/IFR/LIFR) y un perfil atmosférico visual (capas de nubes por altitud, silueta de la ciudad con volcanes, viento y QNH; dibuja lluvia/rayos según el reporte). Con buscador de cualquier código ICAO y accesos a los principales aeropuertos de México. Fuente: aviationweather.gov (NOAA).

Estación remota

Página solo lectura para una segunda estación (p. ej. un Ecowitt GW1100) que envía al mismo servidor. Sus datos se guardan separados de la principal, así que no la afectan; y esta estación no dispara alertas ni publica a redes. Muestra condiciones actuales (temperatura con tendencia a 3 h, humedad y punto de rocío), presión con su tendencia, estadística (mín/prom/máx con selector 24 h / 7 d / 30 d) e histórico (Temp y humedad / Presión). Hay un resumen compacto como tarjeta opcional en Mi tablero.

Vista clásica · Widget · Instalar

La Vista clásica (/) es un tablero sencillo para consulta rápida, unificado con el estilo de /pro. En Widget (/pro/compartir) generas un <iframe> con las condiciones actuales para insertarlo en otra web (eliges unidades, tema y tamaño). Y el sitio es una PWA instalable: en el móvil, «Añadir a pantalla de inicio» lo deja como app (y abre directo la versión completa).

06 — Widget

Widget para tu sitio (/pro/compartir)

Cualquiera puede poner el clima en vivo de la estación en su web o blog con un widget <iframe>. La página Widget para tu sitio es un generador: eliges unidades (°C/°F), tema (claro/oscuro) y tamaño, ves una vista previa y copias el código listo para pegar.

Cómo insertarlo

  1. Ajusta unidades, tema y tamaño hasta que la vista previa te guste.
  2. Pulsa «Copiar código».
  3. Pega el código en el HTML de tu página: en WordPress un bloque «HTML personalizado»; en Blogger/Wix un elemento «Insertar código»; en HTML puro, dentro del <body>.

Ejemplo del código:

<iframe src="https://clima.xe1e.net/embed?units=metric&theme=dark"
  width="360" height="210"
  style="border:0;border-radius:16px;max-width:100%"
  title="Clima CDMX" loading="lazy"></iframe>
  • Ruta base: /embed, con parámetros opcionales ?units=metric|imperial y &theme=light|dark.
  • Muestra: temperatura, sensación y condición, más presión, humedad y viento; se actualiza solo cada minuto.
  • Responsivo (max-width:100%), sin cuentas ni permisos; al hacer clic abre el sitio completo.
07 — Administración

Panel de administración (/admin)

Acceso usuario/contraseña (sesión de 12 h). Diseño compacto en columnas. Permite editar en caliente, sin reiniciar, todo lo configurable.

Wizard de configuración inicial

La primera vez que se accede al panel, un asistente de 5 pasos guía la configuración: 1) Bienvenida, 2) Verificación de estación, 3) Alertas y Telegram (con botón «Probar»), 4) Publicación a redes, 5) Resumen final. Se puede saltar y reacceder más tarde.

Dashboard

Vista general con indicador en tiempo real (pulso verde), historial de alertas de las últimas 24 h (muestra estado activo/resuelto), y acciones rápidas (activar/desactivar alertas, probar Telegram, refrescar). Estado de servicios (InfluxDB, Telegram, WAQI).

Estaciones

Lista de estaciones detectadas con estado (online/offline), última lectura y sensores. Botón «+ Agregar estación» para crear secundarias (nombre + passkey opcional que se autodetecta). Las secundarias pueden eliminarse (con confirmación). Cada fila enlaza a su configuración individual.

Configuración por estación

Nombre/etiqueta, watchdog (activar/desactivar y timeout en minutos). Servicios individuales: activar alertas, publicación a redes y MQTT por estación (secundarias por defecto solo almacenan datos). Sensores WN31 con nombres personalizados (ej. «Sala», «Recámara»).

Alertas

Toggle global y por tipo. Umbrales en línea: Temperatura (alta/baja °C), Viento (sostenido/ráfaga km/h), Lluvia (tasa/diaria mm), Presión (alta/baja hPa). Activar batería baja, sensor perdido, estación offline (minutos) y calidad del aire (AQI/IMECA).

Calibración

Toggle global. Offsets: temperatura (°C), humedad (%), presión (hPa). Multiplicadores: viento y lluvia (factor). Se aplican antes de guardar en InfluxDB.

Publicación

Activar y configurar credenciales de redes públicas: Weather Underground (Station ID + Key), PWSWeather (ID + Password), Windy (API Key), OpenWeatherMap (API Key + Station ID) y CWOP/APRS (Callsign + Passcode + coordenadas). Muestra cuáles redes están activas.

Notificaciones

Toggle de Telegram. Campos: Bot Token y Chat ID. Botón «Probar» para enviar un mensaje de prueba y verificar la configuración.

Integraciones

MQTT / Home Assistant: broker, puerto, topic, usuario y contraseña. Toggle de auto-discovery y prefijo. Indicador de conexión (conectado/desconectado con último error), botón «Probar conexión» (sin afectar la activa) y «Reconectar» para forzar reconexión en caliente. WAQI: token API para calidad del aire (AQI), con enlace para obtenerlo gratis.

Sistema

Información: versión, número de estaciones, última lectura, estado de InfluxDB. Control de calidad (QC): toggle de QC habilitado y filtro de picos. Visor de logs: filtra por nivel (todos/warning/error) con refresco en tiempo real. Enlaces útiles (sitio público, guía, GitHub, API). Stack tecnológico.

Los tokens/claves se muestran enmascarados y si se dejan en blanco al guardar, se conservan. Todo se guarda en /data/settings.json y se aplica al instante. Si ADMIN_USER/ADMIN_PASSWORD están vacíos, el panel queda deshabilitado.

12 — Operación

Operación y mantenimiento

Servidor: VPS Oracle ARM, cuenta PAYG (evita recuperación por inactividad). Dominio clima.xe1e.net tras Cloudflare.

Desplegar / actualizar

cd ~/ecowitt-weather-server-xe1e
git pull
docker compose up -d --build      # --build al cambiar dependencias o imágenes
docker compose ps                 # verificar estado

Los ajustes del panel viven en el volumen receiver-data (/data/settings.json), así que sobreviven a reinicios y reconstrucciones.

Cuando llegue el WS2910

  • Apagar el simulador y limpiar los datos falsos de InfluxDB.
  • En la consola: Weather Services → Customized → Ecowitt, host clima.xe1e.net, ruta /data/report/.
  • Verificar en /api/current y en la web que llegan lecturas reales.
